VeinMiner Doesn't Work on Certain Blocks: 

Step 1: Type in the Command: /veinminer blocklist [Tool to be Used] add [ModName:ItemName] [Metadata (If Any)]*# 

* - The Command will Add that Particular Block to that Particular Tool's Blocklist. In Order to Get the Block's ModName:ItemName Data, get that Particular Block's Data Value, and do /give @p [Data Value] 1 [Metadata (If Any)].
# - Example - Lapis Caelestis Callainus. It's ModName:ItemName Data is ExtraUtilities:greenscreen, and Since the Lapis Caelestis Callainus has a Metadata of 9, the Command would be /veinminer blocklist pickaxe add ExtraUtilities:greenscreen 9.
Help Article Origin: KingWealthyturtle

Worlds Don't Load (They Bring You Back to the Main Menu) / Server is in an Errored State: 

Step 1: Go to Curse\Minecraft\Instances\Universal Interdimensional Escape\scripts
Step 2: Check for Any Folder in the scripts Folder. If Any Folder Exists there, Delete it*#
Step 3: Reload the Game#

* - If No Folder Exists in the scripts Folder, that Means You are Currently Experiencing Some Currently Unknown Issue, and that You Should send me a Private Message, or drop a Comment down at the Comment Section below! (At: http://minecraft.curseforge.com/projects/universal-interdimensional-escape).
# - This is Caused by the CraftTweaker's Incompatiblity with Folders, as well as the Curse App Not Properly Deleting and Reinstalling Content.
Help Article Origin: KingWealthyturtle

 

World has been Overwritten by a Template World:

Step 1: Go to Curse\Minecraft\Backups\Universal Interdimensional Escape\saves
Step 2: Copy the World that You Would Like to Restore
Step 3: Go to Curse\Minecraft\Instances\Universal Interdimensional Escape\saves
Step 4: Delete the World that was Overwritten
Step 5: Paste the World that You Would Like to Restore*

* - You Should Probably Also Rename Your World to Something Like "[Insert Username]'s World 1", etc.
